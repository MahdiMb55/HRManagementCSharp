# راهنمای کارایی

بودجه‌ها روی رایانه اداری متعارف با SSD و 100,000 کارمند سنجیده می‌شوند؛ صدک 95 هدف است.

| عملیات | بودجه |
|---|---:|
| startup تا نمایش shell | 2.0 s |
| اولین render پنجره اصلی پس از bootstrap | 250 ms |
| اولین صفحه فهرست کارکنان | 500 ms |
| پاسخ جست‌وجو پس از debounce | 400 ms |
| بازکردن اطلاعات پایه کارمند | 500 ms |
| ذخیره اطلاعات پایه | 750 ms |
| تعویض تب loadشده | 100 ms |

هیچ query تعاملی همه رکوردها را load نمی‌کند. projection، index، pagination، cancellation و cache lookup ثابت الزامی‌اند. constructor فرم/کنترل I/O ندارد. عملیات بیش از 50 ms در development اندازه‌گیری و عملیات مفید بیش از 100 ms با نام عملیات و مدت ثبت می‌شود. `DevelopmentPerformanceScope` فقط در buildهای Debug فعال است و داده حساس ثبت نمی‌کند.
